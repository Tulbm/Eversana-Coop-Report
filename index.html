The Employer

EVERSANA is a global provider of commercialization services to the life sciences industry. The Value & Evidence team, based in Burlington, supports clients by conducting research and analyzing data to demonstrate the value of medical treatments. This often requires reviewing and synthesizing large volumes of literature and clinical evidence, making automation and AI solutions valuable for improving efficiency, accuracy and cost.

Job Description

During this work term, I served as the sole developer for the Value & Evidence team, maintaining existing applications and building new AI-driven tools to improve literature review and data extraction workflows. My key contributions included:

  • Developing a modular architecture for internal applications, including implementing a RAG pipeline as an external class, enabling multiple apps to share its functionality for document parsing, chunking, embedding, and semantic search.

  • Migrating Shiny apps from R to Python and enhancing them with new capabilities, leveraging libraries such as pandas for data handling, PyMuPDF for document parsing, fuzzy matching for text comparison, and model-related tools like Transformers and tiktoken.

  • Implementing dynamic Pydantic models to generate JSON schemas and customizable prompt templates, improving flexibility and structured interactions with large language models.

  • Experimenting with multiple LLM configurations, using different models and API keys to evaluate performance and cost trade-offs for production scenarios.

  • Managing the full lifecycle of the codebase, including repo structure, environment setup, and thorough documentation to ensure reproducibility and support future app development.

    Goals

Goal 1: Strengthen my ability to develop effective, data-driven solutions for healthcare-related challenges using applied AI and statistical analysis

I achieved this by designing a modular Retrieval-Augmented Generation (RAG) component as an external class, allowing multiple applications to access its features rather than limiting it to one. Over the course of the internship, I iteratively improved older app prototypes by integrating this component, which helped me refine my design skills and better understand how to build scalable solutions.

Goal 2: Become proficient in the tools and frameworks used to develop and evaluate AI models at EVERSANA

I worked extensively with Python Shiny, Azure OpenAI, and supporting libraries, experimenting with multiple LLM configurations across different API keys and incorporating libraries to build feature-rich applications.
To achieve this, I had to dive deep into the documentation of these tools, learning their capabilities and limitations in detail. This hands-on exploration enabled me to confidently apply them in real-world scenarios and troubleshoot complex integration issues.

Goal 3: Improve the clarity and effectiveness of my technical writing in documentation and internal reports

I structured the codebase so that new applications could be added easily in the future, which required detailed documentation on architecture, dependencies, and design patterns to ensure maintainability. This documentation was reviewed by colleagues, who provided positive feedback on both the content and structure and highlighted that it was clear and useful.

This work term significantly strengthened my ability to design and implement scalable, AI-driven applications. I deepened my understanding of large language models and their integration into real-world systems while expanding my skills in Python development, modular app design, and backend/API integration.
